Plugin Information

Distribution of this plugin has been suspended due to unresolved security vulnerabilities, see below.

The current version of this plugin may not be safe to use. Please review the following warnings before use:

  • CSRF vulnerability and missing permission check allow capturing credentials

The XL TestView Plugin integrates Jenkins with XebiaLabs XL TestView

by adding a post-build action to your Jenkins installation:

  • Send test results to XL TestView

Requirements

  • XL TestView: version 1.2.0 or higher.
  • Jenkins: LTS version 1.609.1 or higher.
  • Java: version 7 or higher.

Features

  • Send test results to XL TestView

Configuration

There are 2 places to configure the XL TestView Jenkins Plugin: the global Jenkins configuration and job configuration.

Plugin configuration

At Manage Jenkins -> Configure System you can specify the XL TestView server URL and one or more sets of credentials. Different credentials can be used for different jobs.

Job configuration

In the Job Configuration page, choose Post-build Actions -> Add post-build action -> Send test results to XL TestView. Now you can configure a globbing pattern used to find the test results.

Release Notes

Version 1.2.0 (August 3, 2016)

  • Add the build duration to the metadata sent to XL TestView. This requires XL TestView 1.4.4 or higher. The duration is the time of the build up to the running of the build step.

Version 1.1.1 (February 15, 2016)

Bug fixes

  • Ensure that the 'Mark build as unstable if sending test results fails.' check box works properly.

Version 1.1.0 (November 23, 2015)

Possible breaking changes

  • Build with Java 1.7 and depend on Jenkins 1.609.1. Starting with this version, Jenkins will need to run with Java 1.7.

Bug fixes

  • Do not send extra slashes in the XL TestView REST calls when there are extra trailing slashes in the XL TestView server configuration.
  • Upload test data using chunked encoding.

Improvements

  • Add option to not change the build status when there is a problem communicating with XL TestView.
  • Send user agent identifying the version of the plugin.
  • Explicitly deal with cases where the XL TestView license expired.
  • Improve exception handling for authentication and connection errors.
  • Improve help messages.
  • Improve error messages.
  • Fix IntelliJ project generation.

Version 1.0.1 (June 16, 2015)

  • Fix filtering of test specification sets in the post build step.
  • Small textual improvements.

Version 1.0.0 (June 9, 2015)

  • Upload test results to a test specification.
  • Upload different types of test results to test specifications.
