Jenkins : 2.0 Out-of-the-box experience

Please use epic ticket for comments, feedbacks, etc.

This is a draft document for Jenkins 2.0. None of this is real (yet).

Problem

A revamped new-user experience would help the numerous people who set up Jenkins for the first time.

Description

  • Change how plugin bundling works, don't automatically install plugins just for backward compatibility
  • Some emphasis on Continuous Delivery (include Pipeline Plugin)

References

  • JENKINS-30749 Jenkins should be secure out of the box by default
  • https://youtu.be/kzRR8XR8hu4 Initial working prototype of new experience

This particular feature has been the topic of quite a bit of discussion on the Jenkins Dev list over the past few months. The major remaining issue is determining which plugins should be suggest and/or recommended and by what process that list should be generated. That having been said, comments as to any aspect of this feature are always welcome.
