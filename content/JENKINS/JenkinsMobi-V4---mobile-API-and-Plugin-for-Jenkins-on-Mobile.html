Why another JenkinsMobi ... again ?

Additionally JenkinsMobi is now OpenSource and released under the Apache 2.0 license. You will be driving from now the future of the Jenkins Mobile (r)evolution :-)

Where can I get JenkinsMobi V4 ?

JenkinsMobi V4 Beta is available on Android-OS and is released on a daily basis using JenkinsMobi V3 on its "Default Jenkins Instance".

Why a JenkinsMobi Beta program ?

We want the JenkinsMobi platform driven by its users and from the daily "dog-fooding". Download it, use it on a daily basis and provide feedback and your patches on to improve the platform.

Goals of the JenkinsMobi V4 platform

Goal#1: Improved stability
Client is slim and easily adaptable / portable to other Mobile OSes (iOS, BB10, WP8). Less code means less bugs, memory and more battery life.

Goal#1: Improved stability

Client is slim and easily adaptable / portable to other Mobile OSes (iOS, BB10, WP8). Less code means less bugs, memory and more battery life.

Goal#2: Ease of use

No more complex settings but just one URL and your credentials, easy as using Jenkins from your Browser.

Goal#3: Performance

Content is always cached locally, giving the ability to browse it even off-line. When going on-line, only the differences are downloaded.

Goal#4: API and Plugin integration

JenkinsMobi supports multiple authentications (example: GitHub authentication) and can be extended with new features through Plugin APIs.

How can I provide feedback ?

Post your comments on Jenkins-Dev mailing list using the prefix [jenkinsmobi], Tweet-us @HudsonMobi and #jenkinsmobi

What about new features and updates ?

We are following the Mobile Application Lifecycle principles: new features, software drops and feedback will come to the JenkinsMobi platform itself, through new icons and notifications. Whenever you will see the UPDATE Icon you will get features, info and bug-fixes of every new release.
